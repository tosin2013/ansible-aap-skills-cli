Skills

ansible-aap-skills-cli ships six domain skills targeting the Red Hat CoP Ansible AAP ecosystem, plus one contributor meta-skill for maintaining the repository itself. Each skill is a SKILL.md file placed in your AI assistant’s skills directory, providing domain-specific rules and context for a specific repository or use case.

Skills Overview

Skill Target Repositories ADR
aap-config-structure aap_configuration_template ADR-001, ADR-003

aap-infra-roles infra.aap_configuration ADR-001
ee-yaml-schema ansible-execution-environment, ee_utilities ADR-007
ee-build-workflow ansible-execution-environment ADR-008
ansible-good-practices All repositories ADR-009
skill-research (contributor, opt-in) ansible-aap-skills-cli ADR-011

How Skills Work

  1. Run ./install.sh install — skills are copied to ~/.claude/skills/ and/or ~/.cursor/skills/
  2. Open a target repository in your IDE
  3. Your AI assistant automatically loads the skill context
  4. The AI follows the rules in SKILL.md for every suggestion it makes

The ansible-good-practices skill is a baseline — it is always installed alongside any other skill.

The skill-research skill is opt-in for contributors: ./install.sh install --skill skill-research.
